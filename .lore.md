## Long-term Knowledge

### Architecture

<!-- lore:019e194e-8f9c-7c7a-a11f-3aa4fa840353 -->
* **Gateway-only plugin pattern: no standalone fallback**: Pi plugin follows strict gateway-only pattern: if gateway isn't running, plugin registers no hooks and logs an error. Plugin's sole job: probe/spawn gateway → config hook (disable compaction, register worker agents, redirect provider URLs). All startup tasks live in gateway \`initIfNeeded()\`. Pi plugin exception: \`session\_before\_compact\` hook must remain in plugin because Pi's compaction goes through its extension API, not HTTP. Provider redirection uses \`registerProvider(name, { baseUrl })\`. Gateway agent definitions should include \`LORE\_GATEWAY\_URL\` env var so subagents can also reach the gateway. Only redirect providers whose wire protocol the gateway handles. See \[\[019e1952-398c-753f-b92c-4a5fa5ecf15f]] for full provider compatibility matrix.

<!-- lore:019e1952-398c-753f-b92c-4a5fa5ecf15f -->
* **Pi plugin: which providers can be proxied through the gateway**: Pi plugin gateway proxy compatibility depends on wire protocol. Gateway routes: \`POST /v1/messages\` (Anthropic) and \`POST /v1/chat/completions\` (OpenAI completions). \*\*Proxiable\*\* (redirect baseUrl to gateway): - \`anthropic-messages\`: \`anthropic\`, \`fireworks\`, \`github-copilot\` → \`/v1/messages\` - \`openai-completions\`: \`deepseek\`, \`xai\`, \`groq\`, \`cerebras\`, \`openrouter\`, \`huggingface\`, \`opencode\`, \`opencode-go\` → \`/v1/chat/completions\` \*\*Cannot proxy\*\*: - \`google\`, \`google-vertex\` — native Google SDK - \`amazon-bedrock\` — AWS SDK - \`openai\`, \`openai-codex\`, \`azure-openai-responses\` — use \`/v1/responses\` not \`/v1/chat/completions\` - \`mistral\` — \`mistral-conversations\` protocol, unclear Pi's \`registerProvider(name, { baseUrl })\` overrides base URL; SDK appends correct path automatically.

### Gotcha

<!-- lore:019e18ec-e328-76c4-9c3c-09dbe8d51c6c -->
* **Bun tests leak into production DB when run outside repo root**: Running \`bun test\` outside the repo means \`bunfig.toml\` is not found, preload never fires, and tests silently open the production DB. Fix (now merged to main): \`db()\` throws immediately if \`NODE\_ENV=test\` and \`LORE\_DB\_PATH\` is not set to a temp path — runtime guard rather than relying on preload. \`bunfig.toml\`-based isolation is fragile due to CWD dependency.

<!-- lore:019e1919-9389-7fdc-bb21-5d353df70006 -->
* **Gateway double-compaction bug after structural compaction interception**: Two compaction bugs fixed in gateway pipeline.ts: 1. \*\*Double-compaction\*\*: \`handleCompaction\` must set \`sessionState.messageCount = currCount\` after interception. Without this, the next autocontinue turn re-triggers structural detection. 2. \*\*Subagent compaction leak\*\*: Skip \`isStructuralCompaction\` check for subagent sessions (\`x-parent-session-id\` header present). Structural detection catches post-compaction autocontinue turns; intercepting those for subagents causes the compaction summary to leak as \`task\_result\` to the parent. Pattern-detected (real) compaction requests are still intercepted for subagents — cheaper than upstream API calls. \`detectCompactionRequest()\` in compaction.ts returns typed detection reasons (system-prompt, user-keywords, template-sections); \`isCompactionRequest()\` is a thin boolean wrapper. Diagnostic logging shows detection reason, message count, and tool count.
