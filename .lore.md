* **Gateway recall: Marker-and-Expand strategy — tool injection, interception, and round-trip**: Recall tool injected by gateway pipeline (pipeline.ts), NOT OpenCode plugin. Injection condition: \`modifiedReq.tools.length > 0 && !clientHasRecallTool(tools)\`. Strategy: (1) Response: recall \`tool\_use\` replaced with marker text via \`replaceRecallWithMarker()\`; result stored in \`sessionState.recallStore\`. (2) Next request: \`expandRecallMarkers()\` restores \`tool\_use\`+\`tool\_result\` pairs before forwarding. \`buildRecallFollowUp()\`: assistant message = ORIGINAL \`tool\_use\` block; user message = \`tool\_result\` block with recall results (NOT plain text). \`hasContinuationAfter\` logic: if blocks follow marker, truncates/splices; else prepends tool\_result to next user msg. \`RecallAwareAccumulator\` suppresses recall SSE blocks. MAX\_RECALL\_DEPTH=10. Follow-up requests: \`cacheConversation: false\`. \`finalizeWithRecall()\` closure shared by non-streaming, OpenAI, and openai-responses paths — Anthropic streaming uses \`buildStreamingResponse()\` with recall context passed separately.

(2) \`filterTerms()\`/\`extractTopTerms()\` in \`search.ts\` strip via \`/\[^\p{L}\p{N}\_\s]/gu\` (Unicode-aware) so Turkish words don't split at special letters. Underscore kept for snake\_case.
