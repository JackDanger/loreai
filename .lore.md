## Long-term Knowledge

### Architecture

<!-- lore:019e1c62-3208-7836-a531-f92d1bb20733 -->
* **Conversation import system: providers, detection, extraction pipeline**: Core import system lives in \`packages/core/src/import/\`. Key design: \`AgentHistoryProvider\` interface with \`detect()\`/\`load()\` methods; providers registered in a global registry (\`providers/index.ts\`). Detection scans all providers, returns \`DetectedSession\[]\`. Extraction calls curator LLM sequentially per chunk, deduplicating ops via \`parseOps()\`/\`applyOps()\`. Idempotency via \`import\_history\` table (DB migration v19). Built-in providers: Claude Code (\`~/.claude/projects/\`), OpenCode (SQLite), Aider (markdown), Codex (\`~/.codex/sessions/\` JSONL), Cline (VS Code globalStorage JSON, standard Anthropic MessageParam format, multi-platform, multiple extension IDs), Continue (\`~/.continue/sessions/\` JSON, \`CONTINUE\_GLOBAL\_DIR\` override), Pi (\`~/.pi/agent/sessions/\` tree-structured JSONL using parentId chains \[\[019e1c6c-448f-7b72-b622-4fd885c33522]]). Auto-import triggered in \`lore run\` via \`maybeAutoImport()\`. Copilot Chat skipped (opaque leveldb). Windsurf/Cursor tracked in issue #251.

<!-- lore:019e18ec-6cac-7240-9751-2f5d37b18b31 -->
* **Gateway CJS bundle runs under both Node.js and Bun via polyfill**: Gateway CJS bundle runs under both Node.js and Bun via polyfill: \`dist/index.cjs\` and \`dist/bin.cjs\` are built via esbuild with \`node-polyfills.ts\` shiming \`globalThis.Bun.serve()\` → \`node:http.createServer()\`. An esbuild plugin remaps \`@sentry/bun\` → \`@sentry/node\` at bundle time (esbuild \`alias\` fails for transitive-only deps). Pi and OpenCode plugins use in-process gateway startup via dynamic import — both must externalize \`@loreai/gateway\` in esbuild configs. \`dist/index.d.cts\` only exists post-build; CI typechecks before building. Use variable indirection to prevent tsc from resolving at compile time: \`const mod = '@loreai/gateway'; await import(mod) as any\`.

<!-- lore:019e194e-8f9c-7c7a-a11f-3aa4fa840353 -->
* **Gateway-only plugin pattern: no standalone fallback**: Pi plugin follows strict gateway-only pattern: if gateway isn't running, plugin registers no hooks and logs an error. Plugin's sole job: probe/spawn gateway → config hook (disable compaction, register worker agents, redirect provider URLs). All startup tasks live in gateway \`initIfNeeded()\`. Pi plugin exception: \`session\_before\_compact\` hook must remain in the plugin (Pi compaction goes through its extension API, not HTTP). Provider redirection uses \`registerProvider(name, { baseUrl })\`. Only redirect providers whose wire protocol the gateway handles. See \[\[019e1952-398c-753f-b92c-4a5fa5ecf15f]] for full compatibility matrix. Gateway worker agents need \`LORE\_GATEWAY\_URL\` env var injected in their agent definition so subagents can reach the gateway.

Gateway routes by URL path only.

### Gotcha

<!-- lore:019e1b86-705a-77e7-8b20-92d1c06fe6e3 -->
* **Bun EADDRINUSE error message differs from Node.js format**: Bun's port-in-use error message differs from Node.js: Bun reports 'Failed to start server. Is port X in use?' rather than EADDRINUSE. Catch both with \`/EADDRINUSE/i\` AND \`/is port.\*in use/i\`. \`startGateway()\` in \`start.ts\` is async and returns \`GatewayHandle { owned: boolean, shutdown() }\`. On EADDRINUSE, probes \`/health\` to verify the existing process is a lore gateway — returns \`{ owned: false, shutdown: noop }\` if it is, throws a clear error if not. \`commandStart()\` exits 0 with a message on \`owned: false\`. \`commandRun()\` only calls \`shutdown()\` on signal/child-exit/error when \`owned: true\`. \`probeGateway(baseURL, timeoutMs?)\` uses \`AbortController\` for timeout.

<!-- lore:019e1c27-967c-7eb4-bd0e-afb195823970 -->
* **Bun NAPI crash on process.exit() — use safeExit() via libc \_exit()**: Loading \`fastembed\` (onnxruntime NAPI bindings) into a Bun process causes a C++ panic on \`process.exit()\` because Bun runs NAPI teardown destructors that throw. Fix: use \`\_exit()\` from libc via \`bun:ffi\` to skip teardown entirely. \`packages/gateway/src/cli/exit.ts\` exports \`safeExit(code)\` — uses FFI \`\_exit\` under Bun, falls back to \`process.exit()\` under Node.js or if FFI is unavailable. All gateway exit paths (\`start.ts\`, \`run.ts\`, \`main.ts\`) must use \`safeExit()\`. Also add a double-signal guard in shutdown functions to prevent concurrent \`shutdown()\` calls from spawning duplicate cleanup.

<!-- lore:019e18ec-e328-76c4-9c3c-09dbe8d51c6c -->
* **Bun tests leak into production DB when run outside repo root**: Running \`bun test\` outside the repo means \`bunfig.toml\` is not found, preload never fires, and tests silently open the production DB. Fix (now merged to main): \`db()\` throws immediately if \`NODE\_ENV=test\` and \`LORE\_DB\_PATH\` is not set to a temp path — runtime guard rather than relying on preload. \`bunfig.toml\`-based isolation is fragile due to CWD dependency.

<!-- lore:019e1d28-e23e-7a64-beb9-fa617fac79cd -->
* **cachedProjectPath in pipeline.ts causes cross-project misattribution for compaction/idle work**: In \`packages/gateway/src/pipeline.ts\`, \`cachedProjectPath\` froze the first-ever resolved project path, causing all subsequent compaction runs to be attributed to it. Similarly, \`buildIdleWorkHandler\` in \`idle.ts\` closed over a single \`projectPath\` parameter captured at handler-build time, ignoring \`state.projectPath\`. Fix (implemented): removed \`cachedProjectPath\` entirely from \`pipeline.ts\`; \`handleCompaction\` now calls \`inferProjectPath()\` directly. Changed \`buildIdleWorkHandler\` to remove the \`projectPath\` parameter — reads \`state.projectPath\` at call time instead. The \`process.cwd()\` fallback and greedy generic regex in \`inferProjectPath\` can produce parent-directory paths like \`/home/byk/Code\` → project name "Code" as a remaining risk.

<!-- lore:019e1c27-967f-7ae6-8b22-41332a4ed564 -->
* **embedding.resetProvider() in test teardown causes worker crash via fastembed NAPI**: Calling \`embedding.resetProvider()\` in \`resetPipelineState()\` (test harness teardown) sets \`cachedProvider = undefined\`, allowing pipeline re-init to spawn new fastembed workers. If those workers are alive at process exit, Bun's NAPI teardown crashes (\`workers\_terminated\` mismatch). Fix: move \`embedding.resetProvider()\` to \`shutdown()\` in \`start.ts\` (called only at real process exit via \`safeExit()\`). \`resetPipelineState()\` must NOT call \`resetProvider()\` — preserve the 'fastembed unavailable' cached state so test re-runs don't re-spawn workers.

<!-- lore:019e1c57-aa0b-7f22-b41c-5017f04063b3 -->
* **Gateway DB migrations: SCHEMA\_VERSION is unused — use MIGRATIONS.length**: In \`packages/gateway\`, \`SCHEMA\_VERSION\` is a documentation artifact only. The \`migrate()\` function checks \`if (current >= MIGRATIONS.length)\` — it uses array length, not \`SCHEMA\_VERSION\`. When adding a new migration, only append to \`MIGRATIONS\`; do not update \`SCHEMA\_VERSION\`. The DB test in \`db.test.ts\` hardcodes the expected migration count — update that number whenever a new migration is added (currently 21, with v21 persisting avoided compaction data for idle handler cost tracking).

<!-- lore:019e1919-9389-7fdc-bb21-5d353df70006 -->
* **Gateway double-compaction bug after structural compaction interception**: Gateway double-compaction and subagent compaction bugs (fixed): 1. \*\*Double-compaction\*\*: \`handleCompaction\` must set \`sessionState.messageCount = currCount\` after interception — without this, the next autocontinue turn re-triggers structural detection. 2. \*\*Subagent compaction leak\*\*: Skip \`isStructuralCompaction\` check for subagent sessions (\`x-parent-session-id\` header present) — intercepting autocontinue turns for subagents causes the compaction summary to leak as \`task\_result\` to the parent. Pattern-detected (real) compaction requests are still intercepted for subagents. \`detectCompactionRequest()\` returns typed detection reasons (system-prompt, user-keywords, template-sections); \`isCompactionRequest()\` is a thin boolean wrapper.

<!-- lore:019e1d44-55f7-7fe2-a223-b6f189339e5d -->
* **Historical cost estimates miss 4 of 5 worker buckets — use persisted workerCost instead**: In \`packages/gateway\`, \`computeHistoricalEstimates()\` only re-estimates distillation cost via a heuristic (\`chars/3 \* 3\` input, \`chars/3\` output). It ignores curation, compaction, recall, and warmup worker costs entirely. The real fix: \`idle.ts\` already persists the aggregate \`workerCost\` (sum of all 5 buckets from exact API data) to the DB snapshot on idle. Historical estimates should read this persisted \`workerCost\` directly instead of re-estimating. The heuristic distillation path compounds two layers of approximation vs. exact API token counts from live sessions.

<!-- lore:019e1c6c-4476-7137-ba98-0ba97e5cdd67 -->
* **Import provider message type narrowing: avoid catch-all \`{ type: string }\` variants**: When implementing import providers that parse discriminated union message types (e.g. Codex JSONL, Pi JSONL, Cline MessageParam\[]), TypeScript won't narrow a catch-all \`{ type: string }\` variant properly in switch/if chains. Pattern: use specific string literals for all known types; handle unknowns with an \`else\` branch checking \`(msg as any).type\` rather than a union member with \`type: string\`. This applies to all providers in \`packages/core/src/import/providers/\`.

<!-- lore:019e1b2d-2bbb-7f18-b0cd-8c393aa6693d -->
* **OpenCode plugin spawns gateway via src/index.ts path, but npm publish ships only dist/**: OpenCode/Pi plugins start gateway in-process via \`startInProcess()\` (\`loadConfig()\` + \`startServer()\` from \`@loreai/gateway\`). \`probeGateway()\` runs first; EADDRINUSE falls back to probing. In monorepo, Bun resolves \`@loreai/gateway\` to \`src/index.ts\` via \`'bun'\` export condition; via npm resolves to \`dist/index.cjs\`. \`dist/index.d.cts\` only exists after building — CI typechecks fail. Fix: break static analysis with \`const mod = '@loreai/gateway'; await import(mod)\`. Pi plugin must also externalize \`@loreai/gateway\` in esbuild. \`NODE\_ENV === 'test'\` skips gateway init — export \`startInProcess\`/\`probeGateway\` directly for tests. Use port \`0\` (ephemeral) in tests.

<!-- lore:019e1d1e-cf8b-75d1-a124-d8a1f6e77518 -->
* **OpenCode worker agents (nuum-curator, nuum-distill) flow through full gateway pipeline**: OpenCode sends internal worker agent calls (e.g. \`nuum-curator\`, \`nuum-distill\`) via the \`x-lore-agent\` header through the same gateway endpoint as user turns. These are NOT detected as meta requests (they have large message arrays and system prompts), so they receive full pipeline treatment including temporal storage — causing curator/observer prompts (up to 1.6MB) to pollute the temporal index. Fix: in \`handleConversationTurn\`, check \`input.agent\` against a known worker agent list and skip temporal storage for those turns. Combine with the \`distilled = 0\` FTS filter \[\[019e1d1c-fb4c-7ceb-9890-7da37ed36906]] as defense-in-depth.

<!-- lore:019e1d55-bb26-76a9-a85b-1af0524f48f9 -->
* **Pipeline accumulator only handles Anthropic-format upstream responses — OpenAI upstream is broken**: \*\*Fixed in PR #263.\*\* Pipeline now carries \`effectiveProtocol\` in \`UpstreamResult\` and dispatches to protocol-aware accumulators. Added accumulators for: OpenAI Chat Completions (JSON + SSE), OpenAI Responses API (JSON). Each maps \`prompt\_tokens\_details.cached\_tokens\` → \`cacheReadInputTokens\`. Gateway invariant preserved: internally always returns Anthropic-format; server handlers re-translate to client protocol. If adding a new upstream protocol, extend \`UpstreamResult.effectiveProtocol\` union and add a corresponding accumulator branch — the silent-miss bug pattern is now guarded.

<!-- lore:019e1cca-21d7-7ae9-b874-2c7e8169c22d -->
* **Post-idle compaction fires despite active cache warming (keep-warm sessions)**: In \`packages/gateway\`, \`onIdleResume()\` accepts a 4th param \`skipCompact: boolean\`; sets \`state.postIdleCompact = !skipCompact\`. Gateway passes \`isCacheWarm(state)\` as \`skipCompact\`. \*\*Two bugs in \`isCacheWarm\` for \`/keep\` sessions:\*\* (1) \`isCacheWarm()\` checks \`(now - lastWarmupAt) < ttlMs\` — if the user returns 6.5m after last warmup on a 5m TTL session, it returns false even though the warmer was active. Fix: if \`forceKeepWarm\` is set and at least one warmup occurred, return \`true\` unconditionally. (2) Warmup cadence bug: \`shouldWarm()\` cooldown is \`ttlMs\` (5m) but margin window is 45s — warmer fires every 10m on a 5m TTL, leaving cache expired for ~5m each cycle. Fix: use \`ttlMs - warmupMarginMs\` as cooldown in forced-keep mode.

<!-- lore:019e1c8b-652a-74b9-8f68-763894707dac -->
* **Subagent turns inflate sessionState.messageCount, causing false structural compaction**: In \`packages/gateway\`, \`sessionState.messageCount\` was updated unconditionally for all turns including subagent turns (via \`x-parent-session-id\`). Subagents have independent message arrays that grow to 13+; when the main agent resumes with only 3 messages, the structural compaction detector sees \`prior=13, curr=3\` and incorrectly fires, returning a fake compaction summary instead of forwarding upstream. Fix: guard the \`messageCount\` update in \`handleConversationTurn\` with \`if (!isSubagentTurn)\` — consistent with how EMA and temporal storage already skip subagent turns. Proximity matching is unaffected because subagents are matched via \`x-parent-session-id\` in Tier 1/2 before proximity logic runs.

<!-- lore:019e1d1c-fb4c-7ceb-9890-7da37ed36906 -->
* **Temporal FTS search indexes 90% noise — distilled duplicates + gateway worker prompts**: In \`packages/core\`, temporal FTS \`searchScored()\`/\`search()\` had no \`distilled\` filter, hitting all ~114K rows (~87% already-distilled duplicates, ~3% gateway worker messages, ~0.3% curator/observer prompts up to 1.6MB each). Fix (implemented): added \`AND m.distilled = 0\` to temporal FTS JOINs — existing \`idx\_temporal\_project\_distilled\_created\` index makes this cheap. Vector search already filtered \`distilled = 0\`. A DB migration (v20) purges existing worker noise messages from temporal storage. Gateway worker messages (\`nuum-curator\`, \`nuum-distill\`, \`nuum-consolidate\`, \`nuum-reflector\`) are also filtered at \`store()\` time via \`metadata.agent\` check.

<!-- lore:019e1cd6-05d2-74c5-aea8-fd827a4a45e7 -->
* **vectorSearch() is unscoped — test cleanup must delete all embedding rows**: In \`packages/core\`, \`vectorSearch()\` queries \`knowledge WHERE embedding IS NOT NULL AND confidence > 0.2\` with no \`project\_id\` filter — intentional for cross-project search. Test suites that scope their \`beforeEach\` cleanup to one \`project\_id\` will leak embedding rows into other \`vectorSearch\` tests, causing count mismatches. Fix: the \`vectorSearch\` describe block's \`beforeEach\` must delete ALL knowledge rows with embeddings (\`DELETE FROM knowledge WHERE embedding IS NOT NULL\`), not just rows for its own project. Other suites that don't call \`vectorSearch\` can keep scoped cleanup.

### Pattern

<!-- lore:019e1c62-320d-7f08-8907-cf6ca29eb0d7 -->
* **curator.ts parseOps/applyOps extracted as shared helpers for import pipeline**: \`parseOps()\` and \`applyOps()\` are exported from \`curator.ts\` as named exports so the import extraction pipeline (\`packages/core/src/import/extract.ts\`) can reuse the same op parsing and application logic without duplicating it. When modifying curator op handling, update \`curator.ts\` — the import pipeline inherits the change automatically.
