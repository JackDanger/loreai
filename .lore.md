## Long-term Knowledge

### Architecture

<!-- lore:019e0822-6da2-7164-a809-7f1537b2aa3b -->
* **Cache sacred principle: invalidate only when content changes justify cost**: Core principle for cache management: LTM cache is sacred — treat every invalidation as expensive. Cache busts require Anthropic to write new prefix tokens (cost + latency). Never invalidate preemptively or speculatively. Invalidate only when actual content changes (curation, consolidation, dead-ref cleanup) justify the bust cost. Session-specific invalidations: use \`ltmSessionCache.delete(sessionID)\`. Global invalidations: reserve for project-wide changes. Monitor analytics (hit rate, divergence points) to distinguish legitimate busts (content changed) from unnecessary ones (implementation flaw).

### Gotcha

<!-- lore:019e0794-0778-7157-8029-794369dd0e49 -->
* **Git stash restores unintended changes from multiple developers**: Git cannot recover uncommitted working directory changes: \`git reset --hard\` permanently destroys uncommitted working directory changes never staged/stashed. \`git reflog\` only tracks commits and HEAD movements, not working directory state. Always stage important work or use \`git stash -u\` before risky operations.

<!-- lore:019e0844-75b8-7ad3-8440-884ed69b4651 -->
* **lore emits interactive CLI warnings when started as embedded gateway server**: When OpenCode plugin calls \`startGateway()\` to launch lore programmatically, lore has no agent to run (server-only mode). However, if \`commandRun\` receives no args, it hits agent detection logic and emits interactive CLI advice ("Specify which agent to run: lore run \<command>") to stderr. This warning spam is noise in embedded server scenarios. \*\*Fix\*\*: Add check in lore's \`commandRun\` handler—if stdin is not a TTY AND gateway started programmatically (environment flag or callback), skip agent detection and warnings; go directly to server-only mode announcement.

<!-- lore:019e083b-bd64-7459-9546-4a7e981a15ed -->
* **Pi extension curation callback calls ltmSessionCache.clear() globally**: Pi extension's \`onLtmInvalidated\` callback called \`ltmSessionCache.clear()\` globally, wiping cache for all sessions. \*\*FIXED in PR #160\*\*: Removed \`invalidateLtmCache()\` calls from curation/consolidation/dead-ref cleanup in Pi extension. LTM caches now only refresh on idle-resume when Anthropic cache is already cold (>5min), eliminating mid-session busts from background work. Parallels gateway fix \[\[019e0825-2c3f-7653-9b3f-9c9ffb8c6505]].

* **System prompt layering: host block (1h) + LTM block (no breakpoint) + tools (1h)**: System prompt layering: host block (1h) + LTM block (no breakpoint) + tools (1h): System prompt layering for cache optimization: Anthropic request uses 2-block system array: system\[0] = host prompt with 1h cache breakpoint, system\[1] = LTM knowledge without breakpoint. ### Preference

<!-- lore:019e07a2-a7c4-7a08-a88a-1ddc4b6294e0 -->
* **Auto-merge PRs for infrastructure changes**: Auto-merge PRs for infrastructure changes: User prefers auto-merge enabled on GitHub PRs for infrastructure/optimization work. Pattern: create PR, enable auto-merge with squash option, let CI validate and merge automatically. After merge, pull latest main and apply stashed changes on top for continued development. Reduces manual merge overhead for non-controversial changes with comprehensive test coverage.

<!-- lore:019e0794-07a9-7690-83e4-1a30c814d363 -->
* **User distinguishes between their changes and agent changes in git workflow**: User expects targeted git operations: When user says "stash your changes" they mean only agent's modifications, not all changes. Use \`git status\` and \`git diff\` to identify agent work vs. pre-existing changes. Never use \`git restore\` or \`git checkout\` on files that aren't yours - these are destructive and unrecoverable. Post-merge workflow: always pull latest main first, then apply stashes cleanly with \`git stash pop\`.
