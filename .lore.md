### Gotcha

<!-- lore:019eb4d0-6fd9-7ef5-b22f-0f156eeca4b0 -->
* **cli/main.ts OPTIONS map: undeclared flags silently become booleans under strict:false**: Trap: \`parseArgs\` with \`strict: false\` treats any undeclared \`--flag value\` as \`{flag: true}\` with \`value\` falling into positionals — looks correct because \`--flag=value\` (equals syntax) still works. Fix: every flag used in ANY subcommand MUST be declared in the global \`OPTIONS\` map in \`packages/gateway/src/cli/main.ts\` with its correct type (\`string\` or \`boolean\`). Discovered in PR #696: \`--to\`, \`--project\`, \`--dry-run\`, \`--no-children\`, \`--min-confidence\` were missing, causing \`--to /path\` to parse as \`{to: true}\` with \`/path\` as a positional.

<!-- lore:019ebb4a-397c-76aa-89c8-93110713f3d4 -->
* **formatKnowledge: title-collision within a category produces non-deterministic render order**: Trap: \`formatKnowledge\` sorts entries within each category by title for byte-stable rendering. If two entries share an identical title, the sort returns 0 and their relative order is non-deterministic — text can swap between turns, busting the prompt cache. This is an edge case (duplicate titles are rare) but worth guarding. Fix: add a secondary sort key (e.g. entry id) as a tiebreak after title comparison, mirroring the id-tiebreak already used in the preference fast-path packing loop.
