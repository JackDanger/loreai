* **fetch interceptor: broadened LLM API path patterns**: `packages/core/src/fetch-interceptor.ts` previously had a single narrow pattern (`/v1/(messages|chat/completions|responses)`) that missed aggregator paths (OpenRouter uses `/api/v1/...`, some providers use `/api/...` or `/openai/v1/...`). Fix: 4-pattern array covering `/v1/...`, `/api/v1/...`, `/api/...`, and `/(openai|anthropic)/v1/...`. `matchesLLMApiPath()` iterates the array. `shouldIntercept()` now logs a one-time warning (deduped via `warnedPaths: Set<host+pathname>`) when a remote LLM host is NOT intercepted — operator can extend patterns when a new provider slips through. URL rewriting still uses `lastIndexOf("/v1/")` to extract the canonical `/v1/...` path for the gateway. IPv6 loopback: `URL.hostname` KEEPS brackets (`"[::1]"` not `"::1"`) — both forms must be checked.
