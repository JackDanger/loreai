## Long-term Knowledge

### Architecture

<!-- lore:019e0822-6da2-7164-a809-7f1537b2aa3b -->
* **Cache sacred principle: invalidate only when content changes justify cost**: Core principle for cache management: LTM cache is sacred — treat every invalidation as expensive. Cache busts require Anthropic to write new prefix tokens (cost + latency). Never invalidate preemptively or speculatively. Invalidate only when actual content changes (curation, consolidation, dead-ref cleanup) justify the bust cost. Session-specific invalidations: use \`ltmSessionCache.delete(sessionID)\`. Global invalidations: reserve for project-wide changes. Monitor analytics (hit rate, divergence points) to distinguish legitimate busts (content changed) from unnecessary ones (implementation flaw).

Uses Bun for all build operations (install/typecheck/test/build/pack) not npm. Tarball packing uses \`bun pm pack\` to correctly rewrite \`workspace:\*\` specifiers. Packages under \`packages/\` directory. Publishing uses npm OIDC trusted publishing. To add new CLI: extend tarball creation loop in ci.yml lines 48-73.

<!-- lore:019e07a9-950f-7203-ab27-de6659b511fc -->
* **Plugin role: gateway configuration only, no message handling**: Plugin acts as gateway configuration + fallback only: config hook disables compaction, registers workers, redirects provider baseURLs to gateway. When \`gatewayActive\` true, hooks early-return and skip LTM injection, model limits, idle-resume, message gradient transform, temporal DB storage, calibration, and compaction. Plugin retains: overflow recovery (detects context overflow, triggers distillation + synthetic recovery), active session tracking for \`/compact\`, model metadata capture for worker selection. Plugin-only state: \`ltmDegradedSessions\` (tracks degradation for cache-cost-aware recovery), \`skipSessions\` (filters worker sessions).

### Decision

<!-- lore:019e0825-2c3f-7653-9b3f-9c9ffb8c6505 -->
* **Remove onLtmInvalidated callback from idle handler to scope cache invalidation**: Remove \`onLtmInvalidated\` callback from idle handler to prevent global LTM cache clears. Callback triggered curation/consolidation/dead-ref callbacks to call \`ltmSessionCache.clear()\` globally, wiping cache for all sessions and causing back-to-back busts on unrelated sessions. With callback removed, LTM cache only invalidates on explicit session events (imports, oversized pruning) or when next idle-resume fires for that specific session. Reduces cache bust cascades in multi-session environments.

### Gotcha

<!-- lore:019e0794-0778-7157-8029-794369dd0e49 -->
* **Git stash restores unintended changes from multiple developers**: Git cannot recover uncommitted working directory changes: \`git reset --hard\` permanently destroys uncommitted working directory changes never staged/stashed. \`git reflog\` only tracks commits and HEAD movements, not working directory state. Always stage important work or use \`git stash -u\` before risky operations.

<!-- lore:019e083b-473f-73af-9b61-c8afb8fe486f -->
* **Pi extension has independent LTM injection and cache invalidation paths**: Pi extension duplicates full LTM injection logic in \`onLtmStateChanged\` callback (~line 280 packages/pi/src/index.ts) and maintains separate \`invalidateLtmCache()\` calls for curation/consolidation/dead-refs. When gateway active, Pi's LTM block must early-return to prevent dual injection. Pi also has idle-resume logic (~line 320) that triggers \`onIdleResume\`—this should also early-return when gateway active to avoid redundant cache refresh. These paths were NOT yet updated to respect \`gatewayActive\` flag.

<!-- lore:019e083b-4747-708d-98eb-226c3fb75fd0 -->
* **Pi extension's onIdleResume triggers before gateway's idle logic when both active**: Pi extension's \`onIdleResume\` handler (~line 320 packages/pi/src/index.ts) calls curation, consolidation, dead-ref cleanup, and cache invalidation independently. When gateway active, this causes duplicate idle work: Pi runs its handlers, then gateway runs its own. Must add \`if (!gatewayActive) { /\* Pi idle work \*/ }\` guard to prevent cascading idle operations. Gateway's idle logic already checks gatewayActive, but Pi does not.

<!-- lore:019e0824-437f-7b04-b0df-959b919a6e64 -->
* **Three parallel LTM cache invalidation paths must stay in sync**: LTM cache invalidation logic exists in three parallel code paths: gateway, OpenCode plugin, Pi extension. All maintain identical triggers (curation, consolidation, dead-ref cleanup, imports, oversized pruning). \*\*Gotcha\*\*: Changes to invalidation strategy in one path won't propagate to others. When fixing cache bugs (like the global clear pattern), must coordinate fixes across all three. Pi extension especially: check \`invalidateLtmCache()\` and \`onLtmInvalidated\` callbacks when modifying gateway logic. Recent fix: removed idle-handler global clears from gateway and Pi simultaneously.

* **System prompt layering: host block (1h) + LTM block (no breakpoint) + tools (1h)**: System prompt layering: host block (1h) + LTM block (no breakpoint) + tools (1h): System prompt layering for cache optimization: Anthropic request uses 2-block system array: system\[0] = host prompt with 1h cache breakpoint, system\[1] = LTM knowledge without breakpoint. ### Preference

<!-- lore:019e07a2-a7c4-7a08-a88a-1ddc4b6294e0 -->

Pattern: create PR, enable auto-merge with squash option, let CI validate and merge automatically. After merge, pull latest main and apply stashed changes on top for continued development. Reduces manual merge overhead for non-controversial changes with comprehensive test coverage.

* **User distinguishes between their changes and agent changes in git workflow**: User expects targeted git operations: When user says "stash your changes" they mean only agent's modifications, not all changes. Use \`git status\` and \`git diff\` to identify agent work vs. pre-existing changes. Never use \`git restore\` or \`git checkout\` on files that aren't yours - these are destructive and unrecoverable. Post-merge workflow: always pull latest main first, then apply stashes cleanly with \`git stash pop\`.
