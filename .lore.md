For continuous top-N eviction on quota-capped tables, three cost regimes apply: (1) steady state — zero extra cost, eviction never fires; (2) bounded growth — modest cost, ~1 DELETE + 1 upsert per entry that outranks the floor, proportional to knowledge-creation rate; (3) thrash — potentially unbounded per-cycle churn from (a) value oscillation (confidence decays daily + bumps on reinforce, flipping rows near the cap cutoff in/out every cycle) or (b) multi-device conflicts (two clients evicting each other's rows every 60s indefinitely). Mitigation: per-scope churn budget + circuit breaker in the eviction trigger (see \[\[019f3b79-dfb1-7d1a-999e-e6c6e4b6a4da]]). The dominant baseline cost is already the 60s push+pull polling by every free user — eviction only adds incremental churn on top; lengthening the free-tier sync interval is a separate, easier cost lever.
