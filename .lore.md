## Long-term Knowledge

### Gotcha

<!-- lore:019e18ec-e328-76c4-9c3c-09dbe8d51c6c -->
* **Bun tests leak into production DB when run outside repo root**: Running \`bun test\` outside the repo means \`bunfig.toml\` is not found, preload never fires, and tests silently open the production DB. Fix (now merged to main): \`db()\` throws immediately if \`NODE\_ENV=test\` and \`LORE\_DB\_PATH\` is not set to a temp path — runtime guard rather than relying on preload. \`bunfig.toml\`-based isolation is fragile due to CWD dependency.

<!-- lore:019e1919-9389-7fdc-bb21-5d353df70006 -->
* **Gateway double-compaction bug after structural compaction interception**: Two compaction bugs fixed in gateway pipeline.ts: 1. \*\*Double-compaction\*\*: \`handleCompaction\` must set \`sessionState.messageCount = currCount\` after interception. Without this, the next autocontinue turn re-triggers structural detection. 2. \*\*Subagent compaction leak\*\*: Skip \`isStructuralCompaction\` check for subagent sessions (\`x-parent-session-id\` header present). Structural detection catches post-compaction autocontinue turns; intercepting those for subagents causes the compaction summary to leak as \`task\_result\` to the parent. Pattern-detected (real) compaction requests are still intercepted for subagents — cheaper than upstream API calls. \`detectCompactionRequest()\` in compaction.ts returns typed detection reasons (system-prompt, user-keywords, template-sections); \`isCompactionRequest()\` is a thin boolean wrapper. Diagnostic logging shows detection reason, message count, and tool count.
