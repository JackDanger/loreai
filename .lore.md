## Long-term Knowledge

### Architecture

<!-- lore:019e1c62-3208-7836-a531-f92d1bb20733 -->
* **Conversation import system: providers, detection, extraction pipeline**: Core import system lives in \`packages/core/src/import/\`. Key design: \`AgentHistoryProvider\` interface with \`detect()\`/\`load()\` methods; providers registered in a global registry (\`providers/index.ts\`). Detection scans all providers, returns \`DetectedSession\[]\`. Extraction calls curator LLM sequentially per chunk, deduplicating ops via \`parseOps()\`/\`applyOps()\`. Idempotency via \`import\_history\` table (DB migration v19). Built-in providers: Claude Code (\`~/.claude/projects/\`), OpenCode (SQLite), Aider (markdown), Codex (\`~/.codex/sessions/\` JSONL), Cline (VS Code globalStorage JSON), Continue (\`~/.continue/sessions/\` JSON), Pi (\`~/.pi/agent/sessions/\` tree-structured JSONL). Auto-import triggered in \`lore run\` via \`maybeAutoImport()\`. Copilot Chat skipped (opaque leveldb). The OpenCode plugin's \`reflect.ts\` (plugin-side recall tool) was dead code and has been removed — plugin uses \`tool: {}\` (empty), gateway handles all recall.

<!-- lore:019e18ec-6cac-7240-9751-2f5d37b18b31 -->
* **Gateway CJS bundle runs under both Node.js and Bun via polyfill**: Gateway CJS bundle runs under both Node.js and Bun via polyfill: \`dist/index.cjs\` and \`dist/bin.cjs\` are built via esbuild with \`node-polyfills.ts\` shiming \`globalThis.Bun.serve()\` → \`node:http.createServer()\`. An esbuild plugin remaps \`@sentry/bun\` → \`@sentry/node\` at bundle time (esbuild \`alias\` fails for transitive-only deps). Pi and OpenCode plugins use in-process gateway startup via dynamic import — both must externalize \`@loreai/gateway\` in esbuild configs. \`dist/index.d.cts\` only exists post-build; CI typechecks before building. Use variable indirection to prevent tsc from resolving at compile time: \`const mod = '@loreai/gateway'; await import(mod) as any\`.

<!-- lore:019e194e-8f9c-7c7a-a11f-3aa4fa840353 -->
* **Gateway-only plugin pattern: no standalone fallback**: Pi plugin follows strict gateway-only pattern: if gateway isn't running, plugin registers no hooks and logs an error. Plugin's sole job: probe/spawn gateway → config hook (disable compaction, register worker agents, redirect provider URLs). All startup tasks live in gateway \`initIfNeeded()\`. Pi plugin exception: \`session\_before\_compact\` hook must remain in the plugin (Pi compaction goes through its extension API, not HTTP). Provider redirection uses \`registerProvider(name, { baseUrl })\`. Only redirect providers whose wire protocol the gateway handles. See \[\[019e1952-398c-753f-b92c-4a5fa5ecf15f]] for full compatibility matrix. \*\*Architectural debt\*\*: Pi plugin still directly imports \`@loreai/core\` distillation logic for compaction — prevents fully remote gateway usage. Should be migrated to a gateway endpoint.

* **OpenAI streaming translation: stateful SSE translators for Chat Completions and Responses API**: Pipeline now has true streaming translators for OpenAI protocols (IF-11). Instead of accumulate-then-re-emit, stateful translators consume Anthropic SSE events and emit OpenAI-format SSE events incrementally. Files in \`packages/gateway/src/streaming/\`: \`openai-chat-translator.ts\` and \`openai-responses-translator.ts\`. Each maps Anthropic \`content\_block\_delta\` → OpenAI \`choices\[0].delta\` format. The gateway always processes upstream in Anthropic format internally; translation happens at the egress boundary based on \`effectiveProtocol\`. \[\[019e1d55-bb26-76a9-a85b-1af0524f48f9]]
