## Triage Protocol

### Confidence Weighting

Each finding is assigned a confidence score (0.0–1.0):

| Confidence | Meaning | Verification Required |
|-----------|---------|----------------------|
| 0.95–1.0 | Directly observed in code, mechanically verifiable | None — cite file:line |
| 0.80–0.94 | Strong evidence, minor ambiguity in execution path | Single verification probe |
| 0.60–0.79 | Circumstantial evidence, depends on runtime state | Two verification probes |
| 0.40–0.59 | Plausible from code structure, needs runtime confirmation | Integration test required |
| < 0.40 | Speculative — do not include without escalation | Escalate or discard |

### Verification Probes

For each finding with confidence < 0.95, define a verification probe:

```
Probe VP-NNN:
- Finding: <auditor>.<section> — <title>
- Confidence before probe: <0.0-1.0>
- Probe method: CODE READ | UNIT TEST | INTEGRATION TEST | RUNTIME TRACE
- Probe steps: <specific steps>
- Expected outcome if finding confirmed: <description>
- Expected outcome if finding refuted: <description>
- Confidence after probe (if confirmed): <0.0-1.0>
```

### Cross-Auditor Conflict Resolution

If two auditors reach contradictory conclusions about the same code:

1. Record both findings with their citations
2. Identify the specific line(s) of disagreement
3. The finding with higher confidence and more specific citation wins
4. If confidence is equal, escalate to a focused code read of the disputed section
5. Record the resolution and rationale
